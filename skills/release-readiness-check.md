# Release Readiness Check

## Ziel

Vor Release prüfen, ob Code, Dokumentation, Datenschutz, Security, QA und Betrieb zusammenpassen.

## Prüfen

- Kernfunktionen umgesetzt?
- Akzeptanzkriterien erfüllt?
- Build und Tests erfolgreich?
- Migrationen geprüft?
- Rollback dokumentiert?
- Keine Secrets im Code?
- UI/Brand geprüft?
- Mobile Ansicht geprüft?
- Datenschutzdokumente aktuell?
- TOM/Security aktuell?
- CHANGELOG und README aktuell?
- offene Risiken bewertet?

## Ergebnis

```markdown
## Projektstatus
## Erledigt
## Offen
## Risiken
## Release-Empfehlung
Freigabe: Ja / Nein
```
