# 15 Release Checkliste

## Funktional

- [ ] Kernfunktionen umgesetzt.
- [ ] Akzeptanzkriterien erfüllt.
- [ ] Keine offenen kritischen Bugs.
- [ ] Foto, Plan, Position, Rechte, Chronologie und Audit funktionieren zusammen.

## Technik

- [ ] Build erfolgreich.
- [ ] Tests erfolgreich.
- [ ] Migrationen geprüft.
- [ ] Rollback dokumentiert.
- [ ] Keine Secrets im Code.
- [ ] Queue Worker geprüft.
- [ ] Object Storage geprüft.

## UI / Brand

- [ ] Logo korrekt.
- [ ] Farben korrekt.
- [ ] Typografie korrekt.
- [ ] Mobile Ansicht geprüft.
- [ ] Desktop-Arbeitsansicht geprüft.
- [ ] Accessibility geprüft.
- [ ] Kein horizontaler Scroll.

## Datenschutz / Compliance

- [ ] Datenarten dokumentiert.
- [ ] Datenflüsse dokumentiert.
- [ ] Löschkonzept vorhanden.
- [ ] TOM/Security dokumentiert.
- [ ] Externe Anbieter geprüft.
- [ ] Datenschutzhinweise vorbereitet.
- [ ] KI-Funktionen separat geprüft, falls aktiviert.

## Repository

- [ ] PRs reviewed.
- [ ] `develop` stabil.
- [ ] `main` geschützt.
- [ ] CHANGELOG aktualisiert.
- [ ] README aktuell.
- [ ] Entscheidungen dokumentiert.

## Freigabe

- [ ] QA-Agent.
- [ ] Orchestrator-Agent.
- [ ] Datenschutz- & Compliance-Agent, falls relevant.
- [ ] DevOps-/Security-Agent.
- [ ] Menschliche Projektfreigabe.
