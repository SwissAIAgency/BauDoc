# Spec 14 — Mehrsprachigkeit (i18n)

**Status:** APPROVED · **Bau:** MVP (Infrastruktur + DE) · **Bezug:** D-033
**Source of Truth für:** Internationalisierung/Lokalisierung

## Zweck

Vorsorge für die CH-Amtssprachen (DE/FR/IT) ohne teure spätere Nachrüstung.

## Festlegungen

- **i18n-Infrastruktur von Anfang an:** alle UI-Strings als Übersetzungs-Keys, Backend-i18n (z.B. Laravel Localization).
- Ausgeliefert wird im MVP **nur Deutsch**; FR/IT später ohne Refactoring nachrüstbar.
- Lokalisierung von Datum/Zahl/Dateigröße konsistent (CH-Formate).
- Keine hartkodierten Strings im Code.

## Abhängigkeiten / Verweise

- [[20-mobile-pwa]] · [[13-notifications]] (auch E-Mail-Templates lokalisierbar) · `docs/13_ui_brand_guidelines.md`

## Akzeptanzkriterien

- Keine hartkodierten UI-Strings (lint-/review-prüfbar).
- Sprachwechsel ist technisch ohne Code-Änderung möglich (nur Übersetzungsdateien).
- E-Mail-/Benachrichtigungstexte sind lokalisierbar.

## Offene Punkte

- Zeitpunkt + Umfang der FR/IT-Übersetzung.
