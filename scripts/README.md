# Scripts

Hier liegen später Entwicklungs-, QA-, Datenbank-, Export- oder Wartungsscripts.

Regeln:

- Keine Secrets in Scripts.
- Keine produktiven Daten hart codieren.
- Destruktive Scripts brauchen explizite Freigabe und Schutzmechanismen.
- Scripts müssen im README oder in der Task-Doku beschrieben werden.
